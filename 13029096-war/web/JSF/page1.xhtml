Scopes of Managed Beans

Java Bean

A Java bean is a reusable software component that follows some general specification. A bean is nothing else but POJO(Plain Old Java Object) that defines a public constructor with no parameters and getters and setters method for it's properties. Beans are fundamental to JSF programming.

Managed Beans

Managed beans are regular Java beans that are registered and managed by JSF framework. They work as a model for JSF UI components. Managed bean are directly accessible from a JSF page. In the context of JSF, beans store the state of web pages.

For the convenience of programmers, a JSF container provides different kind of scopes. These scopes typically hold beans and other objects that needs to be available in different components of a web application. In JSF2.2 these scopes can be declared using Annotation as well as using faces-config.xml file.

Application Scoped

A Bean with Application Scoped persists for the entire duration of web application and is shared by all request and sessions.

Session Scoped

A Bean with Session Scoped persists for the time a session is established and until the end the session. A new instance of the bean is created for each session created.

Request Scoped

A Request Scoped bean persists from the start of a request until server sends out a response. A new instances of the bean is created with each request received by the server.

View Scoped

A View Scoped persists as long as user is interacting with the same view(JSF page).

None Scoped

A None Scoped bean persists for a Single EL(Expression Language) evaluation. Bean is destroyed as soon as the EL evaluation is complete.

Custom Scoped

A Managed bean's life time can be defined using Custom Scope. Programmer is responsible for removing Custom Scoped beans from the application.
